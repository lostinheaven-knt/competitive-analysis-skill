# Decision Rules

Use this file when moving from a broad candidate pool to a shortlist.

## Target list sizes
Use these as practical defaults:
- Candidate pool: 10-30
- Filtered set: 5-12
- Core shortlist: 3-7
- Watchlist: any count that is useful

## Inclusion logic
Prefer competitors that overlap on multiple dimensions:
- same or adjacent category
- same target customer
- similar price or commercial model
- similar channel or route to market
- similar workflow solved or use case addressed

The more overlap, the stronger the case for direct competition.

## Exclusion logic
Default to excluding entities with no meaningful overlap in:
- category or product line
- target customer
- route to market
- price segment when price is central to the ask

## Classification
Use clear buckets:
- Direct competitor: strong overlap in problem, customer, and buying context
- Secondary competitor: some overlap, but weaker on one or more dimensions
- Substitute or adjacent player: solves a related need but is not the closest direct rival
- Watchlist: may matter later but evidence is too thin or overlap is still emerging

## Shortlist rules
When choosing the final shortlist:
- include the most decision-relevant competitors, not just the most famous names
- keep at least one note explaining why each name made the shortlist
- remove obvious vanity picks that look large but compete in a different arena
- if a market-defining incumbent is excluded, explain why

## Human review triggers
Pause and reconsider if:
- a highly visible market player is missing
- a shortlisted name has weak overlap with the target
- data quality differs sharply across candidates
- ranking is driven mostly by one noisy metric
- multiple sources use different market definitions

## Final output rule
Do not pretend the shortlist is objective truth. Present it as:
- scope-bound
- evidence-bound
- method-bound

If the scope changes, the shortlist may change.
